# ISO 27001 / ISO 27002 Kontroll-Mapping FreeEPG

## Einleitung

Dieses Dokument mappt relevante Kontrollen der ISO/IEC 27001:2022 und ISO/IEC 27002:2022 auf die konkrete Architektur und Umsetzung von FreeEPG. Für jede Kontrolle werden Umsetzung, Evidence-Pfad, Restrisiko und Prüfstatus dokumentiert. FreeEPG ist ein kleines, self-hosted Open-Source-Projekt ohne dediziertes ISMS; das Mapping dient als Grundlage für gezielte Verbesserungen und externe Audits.

## Geltungsbereich

Geltungsbereich sind alle produktiv eingesetzten Komponenten des FreeEPG-Stacks sowie Entwicklungs- und CI/CD-Prozesse im GitHub-Repository. Kontrollen ohne Relevanz für den aktuellen Projektumfang (z. B. physische Rechenzentren) werden als „nicht anwendbar" markiert.

## Begriffe und Definitionen

| Begriff | Definition |
|---------|------------|
| ISMS | Information Security Management System gemäß ISO 27001 |
| SoA | Statement of Applicability – Erklärung zur Anwendbarkeit von Kontrollen |
| TLS | Transport Layer Security; Verschlüsselung in Transit |
| JWT | JSON Web Token; Session-Format bei NextAuth |
| CI/CD | Continuous Integration/Deployment via GitHub Actions |
| Restrisiko | Verbleibendes Risiko nach Umsetzung der Kontrolle |

## Risiken und Kontrollen

| Risiko | Auswirkung | Eintrittswahrscheinlichkeit | Massnahme | Kontrolle | Nachweis |
|--------|------------|----------------------------|-----------|-----------|----------|
| Unvollständiges ISMS | Audit-Abweichungen | hoch | Schrittweise Kontroll-Umsetzung dokumentieren | Regelmäßiges Mapping-Update | Dieses Dokument |
| Fehlende Backups | Datenverlust bei Volume-Ausfall | mittel | Backup-Prozedur definieren und testen | Restore-Test | **Offener Punkt** |
| Keine Dependency-Scans | Supply-Chain-Angriffe | mittel | `npm audit` in CI ergänzen | CI-Job | `.github/workflows/ci.yml` |
| Admin-Brute-Force | Kompromittierung | niedrig | Rate-Limiting / Fail2ban vor Traefik | Reverse-Proxy-Regeln | **Offener Punkt:** Traefik-Konfiguration extern |
| TLS-intern DB/Redis | Abhörung im Container-Netz | niedrig | Internal network isoliert; TLS optional | Docker network policy | `docker-compose.prod.yml` |

## Pflegeprozess

1. Bei ISO-Normupdates oder signifikanten Architekturänderungen Mapping überarbeiten.
2. Prüfstatus pro Kontrolle bei internen Audits aktualisieren (offen → teilweise → umgesetzt).
3. Offene Punkte in `../audits/audit-dokumentation.md` als Findings tracken.
4. Jährliche Gesamtreview durch Compliance-Verantwortlichen (Rolle organisatorisch festzulegen).
